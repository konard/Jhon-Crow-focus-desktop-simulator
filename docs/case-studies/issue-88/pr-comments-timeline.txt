## üêõ Bug Fix: Document File Upload Fixed

I found and fixed the root cause of the file upload issue.

### The Problem

Several functions in the document handling code used `document` as a parameter name:
- `loadDocToDocument(document, file)`
- `loadDocFromDataUrl(document, dataUrl)`
- `updateDocumentPagesWithContent(document)`
- `updateDocumentThickness(document)`
- `updateDocumentPages(document)`
- `animateDocumentPageTurn(document, direction)`

In JavaScript, this **shadows the global `document` object** (the DOM). So when the code tried to do:
```javascript
const canvas = document.createElement('canvas');  // Line 21592
const content = document.getElementById('interaction-content');  // Line 21358
```

It was actually trying to call these methods on the THREE.js 3D object (the document folder model), not on the DOM! These calls would fail silently since THREE.js objects don't have `createElement` or `getElementById` methods.

### The Fix

Renamed the parameter from `document` to `docObject` in all affected functions. Now:
- `document.createElement()` correctly creates DOM elements
- `document.getElementById()` correctly finds modal elements
- File upload and processing should work properly

### Technical Details

| Line | Issue | Fix |
|------|-------|-----|
| 21260 | `loadDocToDocument(document, ...)` | Changed to `docObject` |
| 21388 | `loadDocFromDataUrl(document, ...)` | Changed to `docObject` |
| 21495 | `updateDocumentPagesWithContent(document)` | Changed to `docObject` |
| 21530 | `updateDocumentThickness(document)` | Changed to `docObject` |
| 21574 | `updateDocumentPages(document)` | Changed to `docObject` |
| 21592 | `document.createElement('canvas')` | Now uses global `document` |
| 21663 | `animateDocumentPageTurn(document, ...)` | Changed to `docObject` |
